CM360 Campaign Workflow

Welcome! If I am on leave, you can use this guide to manage the Djarum campaigns. It covers everything from receiving the brief from planners to generating tags and pulling reports.

Campaign Types Overview

  • CPD (Cost Per Day): Mostly Full-Serve. Involves standard Display/HTML5 creatives.
  • OTT (Over-The-Top): Mostly Tracking Only. We only generate 1x1 tracking tags.
  • Direct Vendor: Tracking only or click-tracker only.

Step 2: Creative Check (CPD Only)

Since OTT is mostly tracking only, this step usually applies to CPD full-serve campaigns. We must ensure HTML5 creatives meet CM360 criteria.

  1. Download the Creative ZIP file from the brief.
  2. Go to the HTML5 Ad Tester.
  3. Upload the ZIP file and check three things:
    • Does the ad render/appear correctly?
    • Is the size correct according to the brief?
    • Is it clickable?

Troubleshooting: Ad is not clickable

If the ad appears but cannot be clicked, confirm with the planner. You may need to inject the manual clickTag into the creative's index.html file. Here is the exact code to use:


<script type="text/javascript">
var clickTag = "%%CLICK_URL_UNESC%%";
</script>

<div id="animation_container" style="background-color:rgba(255, 255, 255, 1.00); width:320px; height:480px">
    <canvas id="canvas" width="320" height="480" style="position: absolute; display: block; background-color:rgba(255, 255, 255, 1.00);"></canvas>
    <div id="dom_overlay_container" style="pointer-events:none; overflow:hidden; width:320px; height:480px; position: absolute; left: 0px; top: 0px; display: block;">
    </div>
    <div id="ad-button" style="width:320px; height:480px; opacity:0; position:absolute; top:0; left:0; z-index:999;"></div>
</div>

<script>
document.getElementById('ad-button').addEventListener('click', function() {
    if (typeof clickTag !== 'undefined' && clickTag) {
        window.open(clickTag, '_blank');
    }
});
</script>

How to use the automation:

  1. Paste your brief data into the respective sheet.
  2. In the Google Sheets top menu, look for "🤖 CM360 Tools" (for CPD) or "🎬 OTT/Tracker Tools" (for OTT).
  3. Click "1. Generate & Sync Taxonomy". The script will automatically format names and ask you for the PO Number if it's missing.
  4. IMPORTANT: Fill in any missing Creative IDs or Landing Page (LP) IDs in the yellow highlighted columns.
  5. Click "2. Generate Format CM360 Import". This will create a separate export file.
  6. Download that export file as Excel (.xlsx) and upload it directly into CM360!

Step 4: Manual CM360 Setup (If needed)

If you need to set up a sudden, urgent campaign manually in CM360 without the import sheet, follow these steps:

CPD (Full Serve) Setup

  • 1 Campaign: Create New -> Name it -> Advertiser: Djarum_ID -> Set Schedule (Start/End date).
  • 2 Landing Page: Click Landing Pages -> Add Name and URL (use Batch if there are many: Column 1 = Name, Column 2 = URL) -> Create.
  • 3 Creative: Click New -> Batch Upload the HTML5/Image creatives.
  • 4 Placement: Click the green "New" button -> Select Site (e.g., Kompas) -> Enter Placement Name -> Set Dimensions -> Save.
  • 5 Ad: Create New -> Select Standard -> Match name to Placement -> Status: Active.
  • 6 Assignment: In the Ad menu, scroll to Creative Assignment -> Edit -> Assign the creative you uploaded. Scroll right to the Landing Page column (pencil icon) -> Assign the specific LP.
  • 7 Placement Assignment: Scroll to Placement Assignment -> Edit -> Select your Placement -> Done -> Save.
  • 8 Tags: Click Tags menu -> Download Tags -> Send to the planner.

OTT (Tracking Only) Setup

The steps are identical to CPD, with three major exceptions:

  • Creative: Do not batch upload. Click New -> Select Tracking.
  • Ad: Still select Standard.
  • Dimensions: Set all dimensions to 1x1.

Step 5: Publisher Previews & QA

Once you send the tags, the publisher will send back a Preview Link to check if the ad is firing properly.

Example Link: https://pubads.g.doubleclick.net/gampad/preview_cookie?...

How to QA the Preview:

  1. Open the preview link in Chrome.
  2. Right-click anywhere and select Inspect (or press F12).
  3. Go to the Network tab.
  4. Paste the Placement ID (from CM360) into the Network filter bar.
  5. Toggle Device Toolbar : Click the device icon next to "Elements" in DevTools. If it's a mobile placement, select a mobile dimension. If desktop, turn it off.
  6. Refresh the page.
    • If the network request shows up, the ad is visible, clicks through to the correct landing page -> It's firing successfully!
    • If it doesn't show up -> Try refreshing again. If still nothing, it is not firing (report back to publisher).
