Manual ClickTag Insertion

If the HTML5 creative fails the click test, you must manually insert the CM360 click macro into the banner's .html file. Follow the 3 steps below.

1 Add Variable in <head>

This defines the destination URL using CM360's unescaped macro.

<script type="text/javascript">
  var clickTag = "%%CLICK_URL_UNESC%%";
</script>

2 Create an Invisible Click Layer

Ensure your banner container has an overlay div (like ad-button) covering the whole banner.

<!-- Example of wrapping Adobe Animate Canvas -->
<div id="animation_container" style="width:320px; height:480px; position:relative;">
    <canvas id="canvas"></canvas>
    
    <!-- ADD THIS LAYER OVER YOUR CANVAS -->
    <div id="ad-button" style="width:100%; height:100%; position:absolute; top:0; left:0; z-index:999; cursor:pointer;"></div>
</div>

3 Add the Event Listener

Place this script at the very bottom, right before </body>, to make the layer trigger the clickTag.

<script>
document.getElementById('ad-button').addEventListener('click', function() {
    if (typeof clickTag !== 'undefined' && clickTag) {
        window.open(clickTag, '_blank');
    }
});
</script>
PHASE 3

Taxonomy on Appscript

A comprehensive tutorial on using Google Sheets AppScript automation to generate naming conventions and CM360 Import formats in just 1-Click.

AppScript Tutorial: CPD (Full Serve)

Use the 🤖 CM360 Tools menu in the Spreadsheet top bar.

Open CPD Taxonomy Sheet
1

Run Taxonomy Sync

Select 🤖 CM360 Tools > 1. Generate & Sync Taxonomy (Active Sheet). The script will automatically read the publisher names and convert them into the standard format.

2

Auto PO Detect & Prompt

If the system doesn't detect a PO number in the existing Campaign name, a pop-up will appear asking for the PO Number. Type or paste the PO number.

3

Fill in the Yellow Columns (Mandatory)

Once the taxonomy is generated, the empty Creative ID and LP ID columns will be automatically highlighted in bright yellow . You must map/create your Creatives & LPs in the CM360 UI first, copy their IDs, and paste them here.

4

Generate & Import to CM360

Once IDs are populated, click 🤖 CM360 Tools > 2. Generate Format CM360 Import. Download the generated spreadsheet as Microsoft Excel (.xlsx). Finally, go to your Campaign in the CM360 UI, click the Import button at the top right, upload the file, and click Save.

AppScript Tutorial: OTT (Tracker Only)

Use the 🎬 OTT/Tracker Tools menu in the Spreadsheet top bar.

Open OTT Taxonomy Sheet
1

Run Taxonomy OTT

Select 🎬 OTT/Tracker Tools > 1. Generate & Sync Taxonomy OTT. This script specifically extracts platforms (like WeTV, Vidio) and campaign context (SeaGames, Mobile) to construct a Smart Contextual Landing Page Logic.

2

Month Aggregation & PO Prompt

The script reads the months included in the raw Campaign column and automatically aggregates them (e.g., JAN_FEB). If the PO is missing, a pop-up prompt will request it.

3

Generate Specific 1x1 Tracker Export

Once the missing IDs (yellow columns) are filled, run 🎬 OTT/Tracker Tools > 2. Generate Format CM360 Import. Download it as Excel and upload it via the Import button in CM360. Unlike CPD, this script automatically sets the Ad & Creative Type to "Tracking" and dimensions to "1x1".

Behind the Scenes of This Script:

The scripts store mapping data in the SITE_MAP & SITE_MAP_OTT variables. If you encounter a new Publisher or Vendor and get a "Mapping Failed" alert, you must add the new publisher and their CM360 Site ID into var SITE_MAP via the Extensions > Apps Script menu first.

PHASE 4

Manual Setup: CPD

If the API/Import fails, or for sudden ad-hoc briefs, follow this UI flow in CM360.

1

Campaign Creation

Click Create Campaign. Enter Campaign Name. Select Advertiser: Djarum_ID. Set the Schedule Start and End Dates.

2

Landing Pages

Click the Landing Page button. For single URLs, enter Name & URL. For multiple, click Batch (requires 2 Excel columns: Name & URL) and paste.

3

Upload Creatives

Click New > Creative > select Batch Upload. Upload your QA-passed HTML5 ZIPs.

4

Setup Placement

Click New > Placement. Select Publisher (e.g., Kompas). Enter Placement Name, set exact Dimensions from brief, and Save.

5

Ad & Assignments

1. Ad Creation: Go to Ad > New > Standard. Use same name as Placement. Set to Active.

2. Creative Assignment: Scroll down to "Edit Creative Assignments". Assign the correct sized creative. Scroll right to attach the Landing Page.

3. Placement Assignment: Assign the Ad to the Placement you just made. Save.

6

Generate Tags

Click the Tags button at the top right, download the Excel tags, and email them to the Planner.

PHASE 5

Manual Setup: OTT (Tracking)

Setup relies exclusively on 1x1 tracking pixels instead of physical assets. Follow this step-by-step UI guide.

What We DON'T Do

Because OTT campaigns are tracking-only, we do not use Batch Upload for HTML5/Video files. The vendor hosts the physical video/banner asset on their end.

What We DO

We generate impression and click tracking pixels in CM360 that the publisher will implement to track performance mapped to our dashboard.

1

Campaign Creation

Click Create Campaign. Enter the OTT Campaign Name. Select Advertiser: Djarum_ID. Set the Schedule Start and End Dates.

2

Landing Pages

Set up your Landing Pages. You can use the Batch entry if you have multiple contextual landing pages mapped out.

3

Create Tracking Creative

Do NOT use Batch Upload. Instead, click New > Creative > Tracking. Name it identically to the placement name.

4

Setup 1x1 Placement

Click New > Placement. Select the Publisher (e.g., Vidio.com). Enter Placement Name. Crucial: Set the dimensions strictly to 1x1.

5

Ad & Assignments

1. Ad Creation: Go to Ad > New > Standard. Set it to Active.

2. Creative Assignment: Under "Edit Creative Assignments", select the Tracking Creative you created. Map the specific Landing Page for this placement.

3. Placement Assignment: Assign this Standard Ad to the 1x1 Placement you just made and Save.

6

Generate Tags

Click the Tags button at the top right, download the Excel tags. You will provide these tracking pixels to the OTT vendor.

PHASE 6

Publisher Preview Verification

How to verify if trackers are firing correctly when publishers send back preview links.

Example Publisher Link

https://pubads.g.doubleclick.net/gampad/preview_cookie?gct=E0btTJ5o5l...&redirect=https://www.vidio.com/watch/...

Verification Workflow

1

Open & Inspect

Open the link in Google Chrome. Right-click and select Inspect (or press F12). Navigate to the Network tab.

2

Toggle Device View

Click the Device Toolbar icon (Ctrl+Shift+M). If it's a mobile placement, select an iPhone/Android layout. If desktop, turn it off.

3

Filter Network

Copy the Placement ID from CM360. Paste it into the "Filter" box inside the Network tab. Refresh the page.

4

Verify Status

Ensure the ad is clickable. Check the Network tab to see if the Placement ID request fired and returned a Status of 200 (OK).

Not Firing?

If the ID doesn't appear or the ad is blank after refreshing, the tracker is not firing. Immediately inform the planner/publisher to fix the implementation.
